# ad-copy

広告コピー(テキスト)を媒体仕様に合わせて生成・検証して納品する。文字数カウントは目視でやらず、必ず `scripts/validate-copy.mjs` で機械検証する(全角=2・半角=1のGoogle方式)。

## 手順

1. **ブリーフの確認**(不足分だけ質問する):
   - 商材/サービスと一番の売り
   - 訴求の切り口(価格・手軽さ・実績・限定性など。指定がなければ複数の切り口に散らす)
   - ターゲットと検索意図(RSAなら対象キーワード群も)
   - 禁止表現・トーンの指定(業界の広告規制があれば)
   - 媒体: 指定がなければ Google RSA。両方も可
2. **媒体仕様**(検証スクリプトが機械チェックする):
   - **Google RSA**: 見出し 半角30字以内×15本、説明文 半角90字以内×4本。見出しの構成目安: ベネフィット系5本・機能/差別化系4本・CTA系2本・信頼/実績系2本・キーワード含有2本。どの見出しとも組み合わさるため、見出し間で意味が重複・矛盾しないこと
   - **Meta**: プライマリテキスト×3本(最初の125字で言い切る)、見出し×3本(40字目安)、説明×3本
3. **生成**: コピーを書き、`copy.json` として保存する(保存先: `~/workspace/experiments/ad-copy/<日付>-<商材slug>/`):
   ```json
   { "headlines": ["..."], "descriptions": ["..."], "primaryTexts": ["..."] }
   ```
4. **文字数検証**(PASS するまで修正して繰り返す):
   ```bash
   node ~/.claude/skills/ad-copy/scripts/validate-copy.mjs <dir>/copy.json --platform google-rsa
   ```
   `--platform meta` も同様。上限ギリギリを狙わず、見出しは25字前後・説明文は80字前後を狙うと調整が楽。
5. **法令・審査チェック**: 全コピーを以下の観点で自己レビューし、該当があれば修正 or ユーザーに指摘する:
   - **景表法**: 「No.1」「最安」「業界初」等の最上級・優良誤認(根拠資料がある場合のみ可、その旨をユーザーに確認)。「必ず」「絶対」「誰でも」の断定
   - **薬機法**(健康・美容系): 効果効能の断定(「治る」「痩せる」「シミが消える」)、医薬品的な表現
   - **その他**: 「無料」の条件明示、二重価格表記、緊急性の煽りすぎ(「本日限り」が常態化する運用はNG)
6. **納品**: `copy.md` に整形して保存し、チャットにも表形式で提示する:
   - 見出し/説明文の一覧(文字数付き)
   - 各コピーの狙い(どの訴求・どのターゲット向けか)を1行ずつ
   - 法令チェックで注意が必要な箇所
   - 入稿方法の一言(RSAは全見出しが総当たりで組み合わさる旨)

## 注意

- 実在の競合名・商標をコピーに入れない。
- 数値実績(「導入1000社」等)はユーザーから提示されたものだけ使う。勝手に創作しない。
- ユーザーの商材理解が浅いまま量産しない。ブリーフが2行以下なら手順1で確認してから書く。
