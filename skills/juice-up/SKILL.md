# juice-up

スピナーだけの退屈な待ち時間を「ショー」に変え、操作の一つ一つに手応えを付ける。
部品のコードは `references/components.md` にあり、コピーして対象プロジェクトに合わせて調整する。

## 手順

### 1. 待ち時間と操作の棚卸し

```bash
grep -rn "animate-spin\|animate-pulse\|読み込み中\|分析中\|生成中\|取得中\|Loading" src --include="*.tsx" -l
```

見つけた待ちを長さで分類する:
- **長い待ち(10秒〜数分)**: AI呼び出し・画像生成・外部APIフェッチ → 最優先で演出
- **ストリーミング出力**: AIが文章を流し込む場所
- **結果表示の瞬間**: 数値サマリー・一覧・完了通知
- **頻繁な操作**: 採用/却下などの判断ボタン、お気に入り、保存

### 2. 部品の移植

`references/components.md` の keyframes を対象のグローバルCSSへ、コンポーネント4種を
`src/components/loading/` へコピーする。Tailwind でないプロジェクトはクラスを素のCSSに読み替える。

### 3. 投入パターン(どこに何を使うか)

| 場面 | 部品 | ポイント |
|---|---|---|
| AI・生成の待ち | `AiThinking` | **ステップ文言はその処理の実際の段階に合わせて書く**(「Metaに接続→集計→画像解決」等)。嘘でも段階が見えると体感が激変する |
| キュー処理中 | `AuroraBar` + 1秒刻みの経過時間表示 | ポーリング間隔と別に `setInterval(1000)` で再レンダーさせ経過秒をライブに |
| ストリーミング出力 | 文末に明滅カーソル | `text && loading` のときだけ `<span class="animate-pulse">` の縦棒を出す |
| 数値の結果表示 | `CountUp` | 消化額・CV・スコアなど「成果の数字」に限定。全部に付けると安っぽい |
| 単発の完了 | `anim-pop-reveal` + `SparkleBurst` | 「直前まで処理中だったもの」だけ光らせる(下記の一度きり管理) |
| 全件完了 | `ConfettiRain` + トースト + `Notification` | 通知の許可はユーザー操作(実行ボタン押下)のタイミングで `requestPermission()` |
| 一覧・カードの表示 | 親に `anim-stagger` | データ確定時にマウントされる要素に付ける。タブの表示切替では再発火しない点に注意 |
| 採用/却下ボタン | ハンコスタンプ(`anim-stamp-in`) | 画像の上に `-rotate-12` の丸枠バッジを重ね、押した瞬間だけアニメクラスを付与 |
| 全ボタン | `button:active { scale(0.95) }` | グローバルCSSに1回書くだけで全体の「触り心地」が上がる |
| トースト | `anim-toast-in` | 下からポヨンと |

### 4. 品質チェックリスト(必須)

- [ ] **すべてのアニメーションを `@media (prefers-reduced-motion: no-preference)` の中に置く**。JS側(CountUp等)は `matchMedia("(prefers-reduced-motion: reduce)")` で即時表示にフォールバック
- [ ] ダークモード: 色はテーマのCSS変数(`var(--color-gray-100)`等)経由にし、固定色はオーロラ系グラデーションなど両テーマで映えるものだけにする
- [ ] **React 19: setStateのupdater関数の中で別のsetStateや副作用を呼ばない**(先に次の値を作り、setStateと副作用を並べて呼ぶ)
- [ ] 「完了演出」は一度きり: 前回ステータスを `useRef(Map)` で持ち、遷移を検知した idを `Set` に入れて `setTimeout` で外す(justDone方式)。永続データから復元した状態では発火させない
- [ ] 永続データに古い形式が混ざる前提でガードする(`STAMP[sel.status]` を引く前に `sel?.status &&` — statusなしエントリでページごと落ちた実例あり)

### 5. 検証

1. `lint` / `tsc` を通す
2. **アニメーションの瞬間は Playwright の要素スクショでは撮れない**(要素が静止するまで待つ仕様)。検証は次で行う:
   - `page.screenshot()`(viewport全体)を150〜200ms間隔で連写
   - `page.evaluate` でアニメクラスの付与とパーティクル要素数をassert
   - 費用のかかる処理(画像生成等)は、状態ファイルやAPIに**擬似データを注入して遷移だけ再現**する(実行はしない)。終わったら必ず原状復帰
3. /screenshot-check でPC・スマホ・ダークの静的崩れも確認
