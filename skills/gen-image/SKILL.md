# gen-image

Codex CLI(ChatGPTサブスク認証)の組み込み画像生成ツール(gpt-image-2)で画像を生成する。API課金は発生せず、ユーザーのChatGPTサブスク利用枠を消費する。

依頼が広告バナー(訴求・案数・入稿サイズがある)なら**バナーモード**、それ以外は**単発モード**。

## 共通: 生成コマンド

保存先ディレクトリで実行する。数分かかることがある(timeoutは7分程度に設定):

```bash
cd <保存先ディレクトリ>
codex exec --skip-git-repo-check --ignore-user-config -s workspace-write --ephemeral \
  [-i <参考画像>] -- "あなたの画像生成ツール(gpt-image-2)を使って、「<プロンプト>」の画像を1枚生成してください。品質は <low|medium|high>、サイズは <WxH> で。カレントディレクトリに <ファイル名>.png という名前で保存し、保存した絶対パスを報告してください。"
```

- 参考画像は `-i`(最大16枚)+プロンプトで「添付画像を参考に」と指示
- プロンプトは被写体・スタイル・構図・色・背景を明示した具体的な指示文にする
- 生成後は必ずPNGをReadで開いて目視確認してからユーザーに見せる

## 単発モード

1. **要件の確認**: 内容(被写体・スタイル・色・雰囲気)、用途、サイズ、保存先。指定がなければ:
   - サイズ: 1024x1024、品質: low(試作) / 用途が本番素材なら medium 以上を提案
   - 保存先: プロジェクト内の作業なら適切なアセットフォルダ、それ以外は `~/workspace/experiments/images/`
2. **生成→確認→報告**: 要望とずれていたら修正プロンプトで再生成を提案する。

## バナーモード(広告用・複数案)

1. **ブリーフの確認**(不足分だけ質問する):
   - 商材/サービス、訴求ポイント(1案1訴求が原則)、ターゲット、トンマナ(参考画像があれば `-i`)
   - サイズ: 指定がなければ Meta 定番の正方形 1080x1080(生成は 1024x1024)。他: 横長 1200x628 相当(1536x1024)、ストーリーズ縦長(1024x1536)
   - 案数: デフォルト3案(訴求の切り口を変える)
2. **消費の確認**: 4枚以上 or high品質はサブスク枠の消費が大きい旨を一言添えて確認する。
3. **文字の扱い**: 日本語コピーの画像内レンダリングは崩れることがある。基本は**文字なしのビジュアル+コピーは別テキストで納品**(Canvaやサイト側で載せる前提)。文字入り希望なら試し、崩れたら文字なしに切り替える。
4. **コピー案**: 各案に見出し(13字前後)+サブコピーを書いて添える。本格的なコピー量産(RSA等)は /ad-copy を使う。
5. **生成**: 保存先 `~/workspace/experiments/banners/<日付>-<商材slug>/`。試作は low、決定稿の清書だけ medium/high で再生成。
6. **提示**: 「画像+見出し+サブコピー+狙い」のセットで提示し、ボツ案の再生成やバリエーション展開を提案する。

## トラブルシューティング

- `Your access token could not be refreshed` → 認証切れ。`codex login` をバックグラウンド起動し、ユーザーにブラウザでの承認を依頼する。
- `codex: command not found` → `/Applications/Codex.app/Contents/Resources/codex` へのシンボリックリンクを `/opt/homebrew/bin/codex` に張る(Codexデスクトップアプリ同梱のCLI)。アプリ自体がない場合は `npm install -g @openai/codex`。

## 注意

- 画像生成ターンは通常のCodex利用より3〜5倍速くサブスク枠を消費する。大量生成の前にはユーザーに一言確認する(目安: low/1024x1024 で約1.8万トークン)。
- 広告用途では実在ブランドのロゴ・人物・競合の模倣を指示しない(審査と権利の両面でNG)。薬機法・景表法に触れそうな表現はコピー段階で指摘する。
