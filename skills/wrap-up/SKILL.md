# wrap-up

セッション終了時に、やり残しを回収して次回すぐ再開できる状態にする。

## 手順

1. **未コミットの回収**: このセッションで触ったリポジトリすべてで `git status --short` を確認する(claude-env / claude-starter / ~/workspace 配下 / その他触ったもの)。
   - 未コミットの変更があれば内容を要約して見せ、コミットする(/ship の安全確認に従う。push は「公開の最終ゲート」なので明示的に確認)
   - 中途半端で捨てるべき変更は、ユーザーに確認してから破棄
2. **メモリ更新**: 進行中のタスク・次の一手・今日決まった方針を、メモリの該当ファイルに追記/更新する(完了済みの古い記述は消す)。
3. **まとめ報告**(簡潔に):
   - 今日できたこと(箇条書き)
   - 未完了・保留(あれば理由も)
   - 次回の最初の一手の提案
4. バックグラウンドで動いているもの(開発サーバー等)があれば停止するか確認する。

## 注意

- 「とりあえず全部コミット」はしない。無関係な変更・実験ゴミは分けて扱う。
- メモリには会話の詳細ではなく「次回の自分が困らない最小限の事実」だけを書く。
