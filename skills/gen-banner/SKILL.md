# gen-banner

/gen-image と同じ仕組み(Codex CLI + gpt-image-2、ChatGPTサブスク枠)で、広告用バナーをサイズ別・複数案生成する。

## 手順

1. **ブリーフの確認**(不足分だけ質問する):
   - 商材/サービス、訴求ポイント(1案1訴求が原則)、ターゲット、トンマナ(参考画像があれば `-i` で渡す)
   - サイズ: 指定がなければ Meta 定番の正方形 1080x1080(gpt-image-2 では 1024x1024 で生成)。他の候補: 横長 1200x628 相当(1536x1024)、ストーリーズ縦長 1080x1920 相当(1024x1536)
   - 案数: デフォルト3案(訴求の切り口を変える)
2. **消費の確認**: 4枚以上 or high品質の場合はサブスク枠の消費が大きい旨を一言添えて確認する。
3. **文字の扱い**: 日本語コピーの画像内レンダリングは崩れることがある。基本は**文字なしのビジュアル+コピーは別テキストで納品**(Canvaやサイト側で載せる前提)。ユーザーが文字入りを希望したら試し、崩れたら文字なしに切り替える。
4. **コピー案の生成**: 各案に見出し(13字前後)+サブコピーを自分で書いて添える。
5. **画像生成**: 保存先 `~/workspace/experiments/banners/<日付>-<商材slug>/` を作り、案ごとに実行:
   ```bash
   codex exec --skip-git-repo-check --ignore-user-config -s workspace-write --ephemeral \
     [-i <参考画像>] -- "あなたの画像生成ツール(gpt-image-2)で「<ビジュアル指示>」の画像を生成。品質 <low|medium>、サイズ <WxH>。<ファイル名>.png で保存し絶対パスを報告。"
   ```
   試作は low、決定稿の清書だけ medium/high で再生成する。
6. **提示**: 全案をReadで開いて確認し、「画像+見出し+サブコピー+狙い」のセットで提示。ボツ案の再生成やバリエーション展開を提案する。

## 注意

- 実在ブランドのロゴ・人物・競合の模倣を指示しない(広告審査と権利の両面でNG)。
- 薬機法・景表法に触れそうな表現(効果断定・最上級表現など)はコピー段階で指摘する。
